# Security Policy

## Reporting a Vulnerability

If you discover a security issue, please report it privately by opening a GitHub security advisory or contacting the repository owner directly.

Please include:

- A clear description of the issue
- Steps to reproduce
- Impact assessment
- Suggested remediation (if available)

## Response Process

- Initial triage within 7 days
- Confirmation and remediation planning after validation
- Public disclosure after patching, when applicable

## Scope Notes

- This repository includes a client-side contact form integration.
- External service misuse risks should be monitored and credentials rotated when needed.
